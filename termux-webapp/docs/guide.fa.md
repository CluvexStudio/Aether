# راهنمای Aether Termux Web

این بسته یک داشبورد وب محلی برای مدیریت Aether روی Termux است.

## شروع سریع

### 1) نصب داشبورد
```bash
cd termux-webapp
chmod +x install.sh
./install.sh
```

### 2) اجرای منوی داشبورد
```bash
aether-web
```

بعد از نصب، همین دستور یک منوی تعاملی باز می‌کند.

### 3) باز کردن در مرورگر
```text
http://127.0.0.1:8787
```

## اولین استفاده

1. در صفحه اصلی روی **Install Aether** بزن.
2. بعد از نصب، یکی از پریست‌ها را انتخاب کن.
3. اگر اینترنتت معمولی است با **ایران / حالت پیشنهادی** شروع کن.
4. اگر QUIC/UDP بسته است، پریست **UDP بسته است** را بزن.
5. اگر شبکه خیلی سخت‌گیر است، **DPI سخت‌گیر** را امتحان کن.
6. تنظیمات را ذخیره کن و **Start Aether** را بزن.
7. لاگ‌ها را پایین صفحه ببین.

## معنی پریست‌ها

### ایران / حالت پیشنهادی
- پروتکل: MASQUE
- ترنسپورت: h3
- نویز: firewall
- اسکن: balanced
- مناسب برای: شروع اولیه و بیشتر کاربران

### UDP بسته است
- پروتکل: MASQUE
- ترنسپورت: h2
- مناسب برای: وقتی h3 یا QUIC وصل نمی‌شود

### DPI سخت‌گیر
- پروتکل: MASQUE
- ترنسپورت: h2
- Fragmentation: روشن
- نویز: gfw
- مناسب برای: شبکه‌هایی که TLS/QUIC را با دقت می‌بندند

### WireGuard سریع
- پروتکل: wg
- مناسب برای: سرعت بیشتر در شبکه‌های کمتر سخت‌گیر

### GOOL پایدار
- پروتکل: gool
- مناسب برای: وقتی WireGuard معمولی پایدار نیست

## تست پراکسی

بعد از اجرا، باید پراکسی روی این آدرس بالا باشد:

```text
127.0.0.1:1819
```

داخل داشبورد روی **Proxy Test** بزن.

یا دستی:

```bash
curl -x socks5h://127.0.0.1:1819 https://www.cloudflare.com/cdn-cgi/trace
```

## اتصال مرورگر

اگر مرورگرت اجازه تنظیم پراکسی جداگانه دارد:

- Host: `127.0.0.1`
- Port: `1819`
- Type: `SOCKS5`
- بهتر است DNS هم از پراکسی عبور کند (`socks5h`)

## گزینه‌های مهم

### Protocol
- `masque`: پیشنهاد اصلی
- `wg`: سریع‌تر ولی کم‌استتارتر
- `gool`: دولایه و مقاوم‌تر

### Scan mode
- `turbo`: سریع‌ترین
- `balanced`: پیشنهاد اصلی
- `thorough`: دقیق‌تر
- `stealth`: آرام‌تر
- `ironclad`: سخت‌گیرترین راستی‌آزمایی

### Quick reconnect
- `on`: اگر آخرین گیت‌وی سالم ذخیره شده باشد سریع‌تر وصل می‌شود
- `off`: همیشه اسکن تازه
- `ask`: رفتاری مشابه حالت CLI

### MASQUE transport
- `h3`: پیش‌فرض، سریع‌تر
- `h2`: وقتی UDP/QUIC محدود است

### Fragmentation
فقط برای h2 است. اگر TLS ClientHello بسته می‌شود روشنش کن.

## لاگ‌ها

لاگ‌ها در این فایل ذخیره می‌شوند:

```text
~/.config/aether-web/aether.log
```

اگر برنامه شروع شد ولی چیزی لود نشد، اول لاگ‌ها را نگاه کن.

## عیب‌یابی

### Aether نصب نمی‌شود
- اینترنت خود Termux را چک کن.
- GitHub روی شبکه‌ات قابل دسترس باشد.
- دوباره روی Install یا Update بزن.

### Start می‌زنم ولی اجرا نمی‌شود
- مسیر باینری را در بخش Advanced چک کن.
- اگر قبلاً دستی نصب کرده‌ای مطمئن شو `aether --version` جواب می‌دهد.
- لاگ‌ها را نگاه کن.

### وصل می‌شود ولی سایت‌ها باز نمی‌شوند
- **Proxy Test** را اجرا کن.
- اگر روی h3 هستی، h2 را امتحان کن.
- اگر روی h2 هستی و باز هم مشکل داری، Fragmentation را روشن کن.
- Scan mode را روی `ironclad` بگذار.

### مدام قطع می‌شود
- پروفایل نویز را سنگین‌تر کن.
- از MASQUE به GOOL یا WireGuard جابه‌جا شو.
- Quick reconnect را روی `on` بگذار.

## اجرای داخل شبکه محلی

برای دسترسی از لپ‌تاپ یا دستگاه دیگر روی همان وای‌فای:

```bash
aether-web --host 0.0.0.0 --port 8787
```

بعد IP محلی گوشی را در مرورگر دستگاه دیگر باز کن.

> این کار را فقط روی شبکه مطمئن انجام بده.
