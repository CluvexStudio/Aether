# Aether Termux Web

یک پنل وب حرفه‌ای برای اجرای **Aether** روی **Termux** و کنترلش از داخل مرورگر.

این ابزار برای وقتی ساخته شده که نخواهی هر بار دستی دستور بزنی و دوست داشته باشی از یک داشبورد تمیز، وضعیت، لاگ‌ها و تنظیمات را ببینی.

## امکانات

- نصب / آپدیت / حذف باینری ترموکس Aether از روی GitHub Releases
- استارت، استاپ و ری‌استارت مستقیم Aether از داخل مرورگر
- نمایش وضعیت زنده، PID، مدت اجرا، نسخه باینری و دستور دقیق اجرا
- ذخیره تنظیمات دلخواه روی خود دستگاه
- پریست آماده برای:
  - MASQUE پیشنهادی
  - حالت UDP بسته / HTTP2
  - DPI سخت‌گیر / Fragmentation
  - WireGuard سریع
  - GOOL پایدار
- مشاهده زنده لاگ‌ها
- تست یک‌کلیکی پراکسی با `curl` روی `socks5h://127.0.0.1:1819`
- راهنمای داخلی فارسی و بخش عیب‌یابی

## نصب روی ترموکس

دو روش داری.

### روش سریع با اسکریپت تک‌فایلی

```bash
chmod +x aether-web-install.sh
./aether-web-install.sh
```

### روش دستی با آرشیو کامل

```bash
cd termux-webapp
chmod +x install.sh
./install.sh
```

بعد از نصب، خود برنامه یک منوی تعاملی **فارسی/English** می‌آورد. اگر بعداً خواستی دوباره همان منو را ببینی:

```bash
aether-web
```

حالا در مرورگر باز کن:

```text
http://127.0.0.1:8787
```

## اگر خواستی از یک دستگاه دیگر روی همان وای‌فای بازش کنی

```bash
aether-web --host 0.0.0.0 --port 8787
```

بعد آدرس IP محلی‌ای که برنامه نشان می‌دهد را در مرورگر باز کن.

> این حالت را فقط روی شبکه‌های محلی مطمئن استفاده کن.

## فایل‌هایی که در زمان اجرا ساخته می‌شوند

```text
~/.config/aether-web/
```

فایل‌های اصلی:

- `config.json` — تنظیمات ذخیره‌شده داشبورد
- `state.json` — وضعیت فعلی / PID
- `aether.log` — لاگ خروجی Aether

## روند سریع استفاده

1. داشبورد را باز کن.
2. یک بار روی **Install Aether** بزن.
3. یکی از پریست‌ها را انتخاب کن یا تنظیمات را دستی تغییر بده.
4. روی **Save settings** بزن.
5. روی **Start Aether** بزن.
6. پراکسی مرورگر یا سیستم را روی `127.0.0.1:1819` تنظیم کن.
7. با دکمه **Proxy Test** مطمئن شو ترافیک عبور می‌کند.

## نکته‌ها

- این داشبورد جای هسته Aether را نمی‌گیرد؛ فقط همان باینری CLI اصلی را مدیریت می‌کند.
- پورت پراکسی خود Aether در UI به‌صورت پیش‌فرض همان `127.0.0.1:1819` است.
- پنل وب عمداً روی `8787` اجرا می‌شود تا با پورت خود Aether تداخل نداشته باشد.
- اگر ریپوی فعلی یک fork از ریپوی اصلی باشد، نصاب سعی می‌کند ریپوی والد را برای آپدیت هسته Aether تشخیص بدهد.
