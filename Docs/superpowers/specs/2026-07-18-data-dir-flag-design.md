# Data Directory Flag (`--data-dir` / `-d`) Design Spec

## Overview
Aether currently provides three separate flags for identity config paths: `--config`, `--wg-config`, and `--masque-config`. When users want to keep all of Aether's generated files in a custom directory (e.g. `~/.config/aether`), they currently have to pass multiple flags or specify paths repeatedly.

This spec adds a single `--data-dir <dir>` (`-d <dir>`) flag and corresponding `AETHER_DATA_DIR` environment variable to set a common directory root for all identity configs, last-connection cache files, and derived sibling configs.

## User Interface & CLI Syntax

### New CLI Flags & Environment Variables
- Flag: `-d <dir>`, `--data-dir <dir>`
- Environment Variable: `AETHER_DATA_DIR`

### Updated Help Usage Output
```text
Config files:
  -d, --data-dir <dir>     base directory for identity and session files
  --config <path>          base identity config path (default aether.toml)
  --wg-config <path>       identity config path for WireGuard
  --masque-config <path>   identity config path for MASQUE
```

## Functional Requirements & Behavior

### 1. Base Directory Precedence & Path Resolution
When `--data-dir <dir>` (or `AETHER_DATA_DIR`) is specified:
1. **Default Config Path**: If `--config` (`AETHER_CONFIG`) is omitted, the base config path defaults to `<data-dir>/aether.toml`.
2. **Explicit Relative Paths**: If `--config`, `--wg-config`, or `--masque-config` is provided with a relative path (e.g., `--config my-config.toml`), the path is resolved relative to `<data-dir>` (e.g., `<data-dir>/my-config.toml`).
3. **Explicit Absolute Paths**: If `--config`, `--wg-config`, or `--masque-config` is provided with an absolute path (e.g., `/etc/aether/custom.toml`), it is used as-is, overriding `<data-dir>`.
4. **Derived Sibling Paths**: All derived files (`*-masque.toml`, `*-lastconn.toml`, `*-secondary.toml`) generated by `derive_sibling_path` inherit the parent directory from the resolved base config path.

When `--data-dir` is not specified:
- Aether preserves existing default behavior (`aether.toml` in current working directory, unless individual flags or env vars are supplied).

### 2. Directory Creation
- When saving configuration files (`config::save`) or last-connection state (`lastconn::save`), Aether checks whether the target file's parent directory exists.
- If the parent directory does not exist, `std::fs::create_dir_all` creates all necessary parent directories before writing the file.
